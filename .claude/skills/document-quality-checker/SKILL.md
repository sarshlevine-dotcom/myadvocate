## Purpose

Final quality gate for all user-facing documents generated by MyAdvocate. Runs three checks in sequence: (1) structural validation, (2) disclaimer verification, (3) **compliance scan for forbidden determinations**. A document cannot leave the system without passing all three.

---

## Instructions

### Step 1 — Structural validation
Verify formatting and completeness:
- Document has proper header (date, sender, recipient block)
- Paragraphs clearly separated, no orphaned placeholder text
- Appropriate length for document type (appeal: 1–3 pages; dispute: 1–2 pages; records request: 1 page)
- All required sections present for document type (see Section Reference below)

**Section Reference by document type:**

| Document type | Required sections |
|---|---|
| Appeal letter | Reference line, statement of appeal intent, factual background, legal/clinical argument, requested remedy, insurer response deadline, contact placeholder |
| Bill dispute letter | Specific disputed charges with amounts, legal basis (No Surprises Act / billing error), requested correction or refund, response deadline |
| Negotiation script | Before-call checklist, opening statement, core ask, supporting points, objection responses, closing, next steps, know-your-rights callout |
| HIPAA records request | Patient identification section, specific records requested, HIPAA citation (45 CFR § 164.524), response deadline (30 days), delivery format preference |
| Ombudsman complaint | Facility identification, incident description with dates, regulatory basis, requested investigation, escalation path |
| State rights summary | Controlling statute citation, plain-language explanation, applicable plan type scope, reporting/escalation path |

### Step 2 — Disclaimer verification
Confirm `legal-disclaimer-enforcer` has been invoked:
- Standard disclaimer block is present at the bottom of the document
- Disclaimer has not been shortened, paraphrased, or moved mid-document
- If missing: **BLOCK delivery** and re-invoke `legal-disclaimer-enforcer` before continuing

### Step 3 — Compliance scan (forbidden determinations)
Scan the full document body for any of the following. Any match is an automatic **FAIL**:

**Forbidden phrases and patterns:**
- Any form of "you should sue" / "consider suing" / "file a lawsuit"
- Any form of "this was illegal" / "they broke the law" / "this is a violation"
- Any form of "you have a case" / "you have grounds" / "you have a strong case"
- Any form of "you will win" / "your appeal will succeed" / "this will be approved"
- Any outcome prediction: probability statements, success likelihood estimates, settlement value estimates
- Any statement implying MyAdvocate is acting as legal counsel, medical provider, or licensed insurance professional
- Any language that contradicts a treating physician's documented recommendation
- Any statement that a specific attorney would take or succeed with the case

**Note:** Citations to federal statutes (ACA, ERISA, HIPAA, No Surprises Act, MHPAEA, FDCPA) are explicitly **permitted** — the compliance scan targets outcome predictions and legal determinations, not citations.

### Step 4 — PII check
Scan document body for any unmasked PII that should have been sanitized:
- Real patient names, DOBs, SSNs, member IDs, addresses, phone numbers, emails
- If found: **BLOCK delivery**, flag the specific field, return to the calling skill to re-invoke `pii-sanitizer`

### Step 5 — Return result
Return one of:
- **PASS** — document cleared all four checks; ready to return to user
- **FAIL — structural** — list specific missing sections or formatting issues
- **FAIL — disclaimer missing** — invoke `legal-disclaimer-enforcer` and resubmit
- **FAIL — forbidden determination** — quote the exact flagged phrase, identify which rule it violates
- **FAIL — PII detected** — identify the field type found; do not quote the actual PII value

## Compliance Notes

- This skill is the `complianceScan` module described in the Repository Deep Dive
- FAIL results must never be silently ignored — the calling skill must fix the issue and resubmit
- Structural FAILs can be fixed by the calling skill; forbidden determination FAILs require human review before delivery
- Crisis resources (mental health, domestic violence, patient advocate hotlines) must never be removed from a document to reduce length
