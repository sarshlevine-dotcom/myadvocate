## Purpose

Design topic cluster architecture that builds domain authority in healthcare advocacy topics through interconnected content.

## Instructions

### Step 1 — Identify parent topic
Select the pillar topic (broad, high-volume term) for the cluster:
- Insurance Appeals (pillar)
- Medical Billing Disputes (pillar)
- Patient Rights by State (pillar)
- Elder Care Advocacy (pillar)
- Specific Condition Coverage Fights (pillar — e.g., mental health, cancer)

### Step 2 — Generate supporting articles
For each pillar, generate 8-15 supporting cluster articles covering:
- Specific denial codes under that pillar
- State-specific variations
- Process-specific guides ("how to file internal appeal with [insurer]")
- FAQ-style articles targeting "People Also Ask" queries
- Case study angles ("what happens when...")

### Step 3 — Map internal linking structure
Define the linking architecture:
- Pillar page links out to all cluster articles
- Each cluster article links back to the pillar page
- Cluster articles link to each other where relevant
- Every article links to the relevant MyAdvocate tool

## Output Format

1. **Cluster Name:** [pillar topic]
2. **Pillar Page Outline:** [H1, H2 structure for the main page]
3. **Supporting Article List:** [8-15 titles with target keywords]
4. **Internal Link Map:** [which articles link to which]
5. **Production Order:** [which to write first for maximum impact]

## Notes

- Start with the pillar page before writing cluster articles
- Google rewards clusters — a strong cluster of 10 focused articles outperforms 10 unrelated articles
- Target: 3-5 complete clusters in Phase 0, expand in Phase 1
