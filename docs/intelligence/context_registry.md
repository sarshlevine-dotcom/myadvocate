# Context Registry — System Overview

**Status:** Active

---

## Purpose

Defines the role of the context registry in organizing system knowledge, state, and intelligence.

---

## What it is

The context registry is a structured data layer (JSON-based) that tracks:
- system components
- task state
- relationships
- execution dependencies

---

## Location

`context_registry/`

---

## Role in system

It acts as a bridge between:
- raw data
- agent logic
- execution workflows

---

## Example uses

- tracking sprint state
- mapping content clusters
- organizing task dependencies
- storing structured knowledge

---

## Relationship to docs

Docs explain systems.

Context registry structures systems.

---

## Critical rule

Registry data must remain:
- structured
- minimal
- purpose-driven

It is not a dumping ground.
